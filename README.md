# E-Commerce Sales Data Analyzer

A Python-based data analytics project developed to extract actionable business insights from retail transaction logs. This repository showcases data cleaning, aggregation, and visualization techniques.

## Overview
This project processes raw e-commerce sales records to answer critical business questions, such as identifying top-performing products, calculating total revenue, and analyzing monthly sales trends to support data-driven decision-making.

## Tech Stack
* **Language:** Python
* **Libraries:** Pandas (Data Manipulation), NumPy (Numerical Analysis), Matplotlib & Seaborn (Data Visualization)
* **Environment:** VS Code / Jupyter Notebook
* **Version Control:** Git & GitHub

## Key Insights & Features
* **Data Cleaning:** Formatted raw object strings into proper datetime structures for reliable time-series analysis.
* **Feature Engineering:** Calculated individual order revenue (Quantity * Price Each) and extracted localized monthly attributes.
* **Data Aggregation:** Leveraged Pandas groupby operations to segment total financial performance by product type and period.
* **Visualizations:** Produced clean, high-contrast behavioral charts detailing revenue generation trends.

## Code Explanation

The core logic of sales_analysis.py follows a standard data engineering and analytics pipeline, broken down into four structural steps:

### 1. Data Ingestion & Setup
The script begins by importing the required ecosystem libraries (pandas, matplotlib, and seaborn) and loading the raw transactional CSV dataset into a Pandas DataFrame.
```python
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns

# Load the dataset
df = pd.read_csv('sales_data.csv')
2. Data Cleaning & Type Casting
Raw data often contains formatting errors. The script explicitly converts the date columns from generic text strings into proper datetime objects. This allows for accurate chronological sorting and time-series extraction.

Python
# Convert Order Date to datetime format
df['Order Date'] = pd.to_datetime(df['Order Date'])

# Extract Month for seasonal trend analysis
df['Month'] = df['Order Date'].dt.month
3. Feature Engineering & Data Aggregation
To calculate total sales performance, a new calculated metric column (Sales) is generated by multiplying the quantity ordered by the individual item price. Python's .groupby() method is then used to aggregate these values by product types and monthly intervals.

Python
# Create a computed revenue feature
df['Sales'] = df['Quantity Ordered'] * df['Price Each']

# Aggregate revenue metrics by month
monthly_sales = df.groupby('Month')['Sales'].sum().reset_index()
4. Data Visualization
Finally, the aggregated metrics are passed into Seaborn and Matplotlib to map out behavioral trends. The script configures high-contrast chart styles, labels axes clearly, and saves the final analytical plots as local image files.

Python
# Generate behavioral sales trend visualization
sns.barplot(x='Month', y='Sales', data=monthly_sales)
plt.title('Monthly Sales Performance')
plt.xlabel('Months')
plt.ylabel('Revenue in USD ($)')
plt.savefig('monthly_sales_trend.png')
